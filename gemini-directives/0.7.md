# 0.7 - Mode de Jeu Turbo

## Objectif
Ajouter un nouveau mode de jeu "Turbo" en plus du mode "Classique" existant pour augmenter la tension et la difficulté.

## Description du Mode Turbo
- **Principe** : Le joueur dispose d'un temps limité pour répondre "Mort" ou "Vivant" pour chaque célébrité.
- **Temps imparti** : 3 secondes par carte.
- **Pénalité** : Si le joueur ne répond pas dans le temps imparti :
    - Il perd une vie immédiatement.
    - La carte actuelle est considérée comme une erreur (ou passée, à définir, mais perte de vie actée).
    - Le jeu passe à la carte suivante (ou Game Over si plus de vies).

## Changements Fonctionnels

### 1. Sélection du Mode
- Au démarrage (écran d'accueil), l'utilisateur doit pouvoir choisir entre :
    - Mode Classique (fonctionnement actuel).
    - Mode Turbo.
- Ajouter des boutons ou un sélecteur clair sur la page d'accueil.

### 2. Gameplay (Mode Turbo)
- **Timer** :
    - Un timer de 3 secondes se déclenche dès l'affichage de la carte.
    - Doit être visible par le joueur (barre de progression dégressive ou compte à rebours numérique).
    - Le timer se met en pause/reset lors des transitions (feedback, game over).
- **Timeout** :
    - À 0 seconde, déclencher l'action "Mauvaise réponse" ou une action spécifique "Trop lent".
    - Feedback visuel spécifique "TROP LENT !" ou similaire.
    - Retrait d'une vie.

### 3. UI/UX
- **Accueil** : Design des boutons de choix de mode cohérent avec la charte "dark/néon".
- **Jeu** : Ajout de l'élément visuel du timer (ex: barre rouge qui diminue rapidement).

## Impacts Techniques
- Modification de la classe `Game` pour gérer l'état `gameMode` ('classic' | 'turbo').
- Ajout d'une mécanique de `setInterval` ou `requestAnimationFrame` pour le timer dans `Game` ou `main.js`.
- Gestion du nettoyage du timer (clearTimeout/Interval) entre les tours pour éviter les bugs.
